# GemifyChat

A ChatGPT-style AI chat application built with **Next.js 16**, **Convex**, **Clerk**, and **Google Gemini**.

## Features

- Authentication with Clerk (Email + Google)
- Real-time streaming responses from Gemini 1.5 Flash
- File upload (images, documents)
- Chat history search, rename, delete
- Dark / light mode
- Responsive (mobile + desktop)

## Tech Stack

| Layer     | Tool                              |
| --------- | --------------------------------- |
| Framework | Next.js 16 (Turbopack, App Router)|
| Language  | TypeScript                        |
| Backend   | Convex (realtime DB + functions)  |
| Auth      | Clerk                             |
| AI        | Google Gemini 1.5 Flash           |
| Styling   | Tailwind CSS + Radix UI           |
| State     | Zustand                           |

## Prerequisites

- Node.js **20.9+** and npm
- A [Clerk](https://clerk.com) account
- A [Convex](https://convex.dev) account
- A [Google AI Studio](https://aistudio.google.com/app/apikey) API key

## Setup — step by step

### 1. Clone and install

```bash
git clone https://github.com/sankha4567/Chatbot.git
cd Chatbot
npm install
```

### 2. Create your env file

```bash
cp .env.example .env.local
```

Leave the values as placeholders for now — you will fill them in during the next steps.

### 3. Set up Clerk

1. Go to [dashboard.clerk.com](https://dashboard.clerk.com) and create a new application.
2. Enable **Email** and **Google** sign-in methods.
3. From **API Keys**, copy the publishable and secret keys into `.env.local`:
   - `NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY`
   - `CLERK_SECRET_KEY`

### 4. Create a Clerk JWT template for Convex

1. In Clerk dashboard → **JWT Templates** → **New template** → name it exactly `convex`.
2. Use this claim set:
   ```json
   {
     "aud": "convex",
     "iss": "{{env.CLERK_JWT_ISSUER_DOMAIN}}",
     "sub": "{{user.id}}"
   }
   ```
3. Copy the **Issuer** URL shown on that template page and put it in `.env.local` as:
   ```
   CLERK_JWT_ISSUER_DOMAIN=https://your-subdomain.clerk.accounts.dev
   ```

### 5. Set up Convex

Run the Convex CLI — it will log you in, create/link a project, and fill in `CONVEX_DEPLOYMENT` + `NEXT_PUBLIC_CONVEX_URL` in `.env.local` automatically:

```bash
npx convex dev
```

Leave it running in that terminal, or stop it after the first sync (`npm run dev` restarts it).

Also set this in `.env.local` (Convex prints the `.site` domain on first sync):

```
NEXT_PUBLIC_CONVEX_SITE_URL=https://your-deployment-name.convex.site
```

### 6. Wire Clerk into Convex

1. In the [Convex dashboard](https://dashboard.convex.dev), open your project → **Settings → Authentication**.
2. Add a provider → **Clerk** → paste the same `CLERK_JWT_ISSUER_DOMAIN` value from step 4.
3. Save. `convex/auth.config.ts` in this repo reads the domain from the environment, so nothing else to edit.

### 7. Get a Gemini API key

1. Go to [Google AI Studio → API keys](https://aistudio.google.com/app/apikey).
2. Create a key.
3. Paste into `.env.local` as `GEMINI_API_KEY`.

### 8. Run the dev server

```bash
npm run dev
```

This runs Next.js (`next dev --turbopack`) and `convex dev` **concurrently** via the `dev:all` script in `package.json`.

Open [http://localhost:3000](http://localhost:3000) and sign up.

## Deployment

### Vercel (frontend)

1. Push this repo to GitHub.
2. Import it in Vercel.
3. Add every variable from `.env.example` in Vercel → Project → Settings → Environment Variables.
4. Deploy.

### Convex (backend)

```bash
npx convex deploy
```

Then update `NEXT_PUBLIC_CONVEX_URL` and `NEXT_PUBLIC_CONVEX_SITE_URL` in Vercel to the production Convex URLs.

## Troubleshooting

- **Sign-in redirects forever** — Clerk JWT template is not named exactly `convex`, or the Convex auth provider is not configured (step 6).
- **`Cannot find module convex/_generated/...`** — run `npx convex dev` once to generate it.
- **`401` from `/api/chat`** — missing or invalid `GEMINI_API_KEY`.
- **Convex CLI asks for login every run** — that's normal the first time; subsequent runs use the cached credential.
